Danger!

Privly is being actively developed and improperly changing these settings could potentially endanger your data. Do not modify these settings without understanding the consequences.

User Whitelist

The user whitelist specifies which domains can be injected into the host page without any user interaction. This extension currently uses remote code in the injected application, so adding untrusted domains to this list could lead to spoofing. In short, don't use this option unless you are a developer.

This list assumes that domains will be entered without protocol, and with only commas separating the domains.

Examples:

  • priv.ly
  • priv.ly,privly.org
  • subdomain.privly.org,priv.ly,privly.org



Content Servers

Set which domain you upload your content to when generating new links directly from the extension.

These are the recommended servers during development, but if you select "other" you will be able to assign it to be any domain you like. Please be certain that the custom domain is trustworthy and keeping up with system development.
