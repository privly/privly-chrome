Privly Extension Options: Developers Only

User Whitelist

The user whitelist specifies which domains can be injected into the host page without any user interaction. This extension currently uses remote code in the injected application, so adding untrusted domains to this list could lead to clickjacking. In short, don't use this option unless you are a developer.

This list assumes that domains will be entered without protocol, and with only commas separating the domains.

Examples:

  • priv.ly
  • priv.ly,privly.org
  • subdomain.privly.org,priv.ly,privly.org
